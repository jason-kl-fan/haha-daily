2026 年 3 月 13 日｜Storage / Server

存儲、伺服器、企業基礎設施軟硬體與供應鏈。

今天這區的主線不是單一新品，而是 **AI 基建正在往更模組化、更跨區域、也更偏向營運可控** 的方向推進：從資料中心落地、雲端命名與跨帳號管理，到企業級 SSD 與 AI 儲存展示，都在補同一件事——把基礎設施擴張變得更能真的用。

快速導讀 / TL;DR

  • **資料中心繼續往前蓋**：Crusoe、Microsoft、QTS、RadiusDC 今天的消息都在指向更快部署、更多區域與更大的園區版圖。
  • **雲端基礎設施更重視治理與擴展細節**：AWS 把 S3 命名空間、OpenSearch 跨帳號分析，以及新一代通用實例區域擴張都往企業可操作性推。
  • **企業儲存正被 AI 重新定價**：KIOXIA 展示 AI 儲存路線，TrendForce 則顯示 enterprise SSD 營收在推論與升級需求下大幅跳升。

Crusoe 推出 Edge Zones，主打用自製模組化 AI 資料中心把算力帶到客戶端

**AI 基建開始從超大型園區，延伸到更可搬運、可就地部署的模組化形態。**

根據 3/13 的 DCD 與 Gemini 搜尋彙整，Crusoe 新推的 Edge Zones 以自家 Spark 模組化資料中心為核心，瞄準低延遲與主權部署需求，之後還會補上液冷版本。重點不是再蓋一座 megacampus，而是把 AI 基礎設施做成更容易快速落地的產品。

Microsoft 在德國 Bergheim 動工新資料中心，延續當地 AI 與雲端擴建

**歐洲資料中心擴張現在不只是容量問題，也是在搶 AI 與數位主權的地理位置。**

DCD 3/13 報導指出，Microsoft 在德國 Bergheim 的新資料中心正式動工，屬於其德國 AI 與雲端擴建計畫的一部分。這代表 hyperscaler 競爭焦點已經從單純多買 GPU，轉向誰能更快把區域性基礎設施、合規與供應落地。

QTS 目標在喬治亞推進兩座大型資料中心園區

**美國資料中心戰線還在往電力與土地更充足的州持續外擴。**

DCD 3/13 指出，QTS 正在喬治亞推進兩個大型 data center campus 計畫，其中包含多棟機房與配套變電設施。這類消息的關鍵不是單一專案本身，而是 cloud 與 AI 工作負載對區域電力、土地與長期擴建能力的需求正在越來越前置。

RadiusDC 進軍亞利桑那，收購 phoenixNAP 鳳凰城機房與 colocation 業務

**資料中心市場不只在新建，也在透過併購更快卡位熱門區域容量。**

DCD 3/13 報導，RadiusDC 將收購 phoenixNAP 在鳳凰城的資料中心與機櫃代管業務，並把它納入後續擴建版圖。對企業基建市場來說，這反映出成熟設施、互連能力與可立即擴容的土地權利，本身就是稀缺資產。

Amazon S3 推出 account regional namespaces，企業可保留自己的區域 bucket 命名空間

**雲端儲存競爭已經細到『命名治理』這種會直接影響大規模營運的層次。**

AWS 3/12 宣布，企業可以在自己的帳號與區域命名空間內建立 S3 general purpose buckets，不再需要全球唯一名稱大海撈針。對多團隊、多客戶或多資料集架構來說，這讓 bucket 命名、政策控管與跨區一致性都更容易落地。

Amazon EC2 M8i 與 M8i-flex 擴至更多區域，補強通用型伺服器工作負載

**不是所有基建升級都在追 GPU；通用型 compute 的區域鋪設仍是企業 IT 的基本盤。**

AWS 3/12 把採 Intel Xeon 6 的 M8i 與 M8i-flex 擴到愛爾蘭與倫敦，主打較前代更好的價格效能與記憶體頻寬。這類更新看起來不花俏，但對 web、資料庫、企業應用與虛擬桌面等常見 server workload 其實很實用。

OpenSearch UI 支援跨帳號存取不同網域資料，集中觀測與分析更容易

**企業基建正在把『資料留在原地、分析跨帳號進行』當成標準能力來做。**

AWS 3/12 宣布 OpenSearch UI 現可直接從單一介面查詢不同帳號中的 OpenSearch domains，不必切換 endpoint，也不用先複製資料。對 observability、搜尋與安全分析團隊來說，這能明顯減少資料管線與多帳號治理的摩擦。

來源：blogwatcher RSS + AWS What's New · 原文連結

KIOXIA 在 GTC 2026 端出 AI 儲存展示，從向量資料庫到 245.76TB QLC SSD 一起上桌

**AI 儲存不再只是『更快的 SSD』，而是整條資料管線如何減少 DRAM 壓力與加速檢索。**

KIOXIA 3/12 預告在 GTC 展示 AiSAQ、CM9 enterprise NVMe SSD、100M IOPS 等級模擬器與 245.76TB LC9 QLC SSD。這些訊號很明確：企業儲存廠商正在把自己定位成 AI pipeline 的關鍵組件，而不只是後端容量供應商。

TrendForce：前五大 enterprise SSD 廠商 4Q25 營收季增逾五成，AI 推論需求成主因

**企業級 SSD 已經從成本項目，變成 AI 基建擴張時最直接被重新定價的部位之一。**

Gemini 搜尋彙整 3/13 的 TrendForce 調查指出，全球前五大 enterprise SSD 廠商 2025 年第四季合計營收季增 51.7%，其中 SK 集團增速領先。背後原因不只是一波伺服器升級，而是 AI 推論與高容量 SSD 需求正在同步拉高。
